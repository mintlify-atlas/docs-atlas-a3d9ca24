# Welcome to ELI NoSQL

ELI NoSQL is a custom NoSQL database language with a full-featured compiler and integrated development environment. Built with Java, JFlex, and CUP, it provides a complete solution for learning compiler construction while working with a practical database system.

## What is ELI?

ELI (Educational Language Interpreter) is a domain-specific language designed for NoSQL database operations. It features:

- **Custom Syntax** - Intuitive commands for database operations (DATABASE, TABLE, ADD, READ, UPDATE, CLEAR, EXPORT)
- **Rich Data Types** - Support for int, float, string, bool, array, and object types
- **Powerful Filtering** - Filter expressions with relational and logical operators
- **JSON Persistence** - Automatic serialization and deserialization of database state
- **Full IDE** - Swing-based editor with syntax highlighting, drag-and-drop, and real-time error reporting

## Architecture Overview

ELI is built on industry-standard compiler tools:

- **JFlex** - Generates the lexical analyzer from `.jflex` specification
- **CUP** - Generates the parser from `.cup` grammar definition  
- **Custom Interpreter** - Executes parsed commands and manages in-memory database
- **JSON Persistence** - Stores database state using `json-simple` library

## Quick Example

Here's a taste of what ELI code looks like:

```eli example.code
## Create a database with JSON persistence
database university {
  store at "university.json";
}
use university;

## Define a table schema
table students {
    id: int;
    name: string;
    age: int;
    gpa: float;
}

## Insert records
add students {
    id: 1,
    name: "Alice Johnson",
    age: 20,
    gpa: 3.8
};

## Query with filters
read students {
  fields: name, gpa;
  filter: gpa >= 3.5;
};

## Update records
update students {
  set: gpa = 4.0;
  filter: name == "Alice Johnson";
};

## Export results
export "top_students.json";
```
